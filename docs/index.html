    txtprint // jeremy jacob

    November 5th

    week 0

    This class period I began work on this website. That includes the text for the #about section, setting up the navigation bar and #typeset page. I later finished the updates page at home. I decided to make this site a one page design, using URL fragments for navigation. This uses JavaScript to pipe HTML to the content div, based on the current URL fragment. This allows for a more fluid and smooth experience, as well as less data being retrieved on navigation. Using fragments retains the use of back and forward buttons in Chrome, whereas using a variable to keep track of the current page does not (this is what I was using before). My goal is to finish the site design by then end of this month, so I'm already on the right track to do that. As of writing this, the docs section is still entirely incomplete and the typeset & updates sections need work. I also plan to buy a domain and host this site by then end of November. I will update this page every week, so that one can see my steps and how long they took. The final code and assets will be posted in #docs.

    txtprint

    a project by Jeremy Jacob for Full Steam Ahead

    AE Wright Middle School, Robotics 8th Grade

    txtprint is a 3d printed portable thermal printer designed for students and classroom use. It uses the Chrome serial API to communicate with an Arduino-style board. I made it because from personal expirience, being able to quickly print conclusions or other text in class is an invaluable asset. I was a group member in a previous FSA project, Classmate. That project also used an Arduino, so I have expirience in that area. As for web coding, it is largely new to me. This is the first full website I have built enitely from scratch. It uses the WebUSB protocol in Chrome to connect and communicate with a Feather 32u4 board. It also has a built-in bluetooth chip, so in the future I can add support for BT and USB connections.
